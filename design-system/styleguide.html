Handover

Adopt in three moves.

Vendor tokens.css, import once, reference var(--ds-*). Tailwind users bridge the vars in config.

/* app globals — import the shared tokens once */
@import "../design-system/tokens.css";
body { background: var(--ds-bg); color: var(--ds-text); font-family: var(--ds-font-sans); }

// tailwind.config.ts — bridge tokens so utilities theme automatically
colors: {
  brand:   "var(--ds-brand)",   // Mint #0D9488
  surface: "var(--ds-surface)",
  "accent-1": "var(--ds-accent-1)", // Analysis
  "accent-2": "var(--ds-accent-2)", // MSAT
}
  • Map, don't add. Product entities take a generic accent slot (1–4); status always uses semantic tokens.
  • Both themes. Honor prefers-color-scheme and :root[data-theme]; never encode meaning by color alone.
  • Govern by version. Edit tokens.css + tokens.json together, bump --ds-version; a change is DS-worthy only if both products benefit.
